Table

Technical Description

The HTML and CSS implementation of the table page follows a structured approach, ensuring both functionality and aesthetics. The HTML structure is built using semantic elements such as header tag, main tag, and footer tag to enhance readability and accessibility. The navigation bar is constructed using a nav tag element with anchor "a" tag tags for page links, styled for a responsive design with a hamburger menu for mobile users. The main content is introduced with a div tag containing a heading to provide context. The table structure uses table tag, thead tag, tbody tag, tr tag (table rows), and th and td tags (table headers and data cells) to organize conservation-related information efficiently. Rowspan and colspan attributes ensure a clean layout, merging specific cells to improve readability. Each region is visually enhanced with emojis and an image-based table header to engage users. Hyperlinks within the table direct users to external conservation programs. The CSS styles, stored in external stylesheets, define layout and design aspects. The grid and flexbox models are utilized for structuring sections dynamically. The navigation bar and table elements use hover effects, background colors, and padding/margins to enhance user interaction. The footer section includes a logo, a slogan, descriptive text, quick links, and social media icons, structured with div containers for a balanced design. The page effectively combines content and styling to create an informative, visually engaging, and responsive user experience while maintaining accessibility and usability.

Accessibility

Accessibility Features in the Table Page

  • Semantic HTML: Used header, main, footer, and nav tags for better screen reader support.
  • Table Structure: Added th tag with scope="col"/row", caption tag for description, and ARIA labels where needed.
  • Text & Contrast: Ensured high contrast, readable fonts, and avoided color-dependent information.
  • Descriptive Links & Alt Text: Used meaningful anchor text and alt attributes for images.

These features enhance accessibility, ensuring usability for all users.

Content Page - Deforestation

Technical Description

The webpage on deforestation is structured using semantic HTML elements for clarity and accessibility, with a header containing navigation and a hamburger menu for responsiveness. A main container with a sidebar allows easy in-page navigation, while the main section presents structured content with headings, paragraphs, and images for better readability. The footer includes additional links and social media icons. CSS enhances the design with a fixed background image, ensuring a visually engaging layout. The sidebar has a distinct background color for contrast, and the content sections are styled for consistency. Flexbox is used to arrange the sidebar and content efficiently. The design maintains a responsive layout, ensuring usability across devices.

Accessibility

Accessibility Features in the Content Page

  • Semantic HTML: Used header, main, aside, footer, nav and section tags for better screen reader navigation and content structure.
  • ARIA Attributes: Added ARIA attributes to the hamburger menu to indicate its expanded/collapsed state for screen readers.
  • Keyboard Navigation: Sidebar navigation enables users to jump to sections without excessive scrolling, improving usability for keyboard users.
  • Contrast and Readability: Ensured distinct background colors for the sidebar, meeting contrast ratio standards for better text readability.
  • Alternative Text: Provided meaningful alt attributes for images to aid visually impaired users.
  • Responsive Design: Used flexbox layout to adapt the page to different screen sizes, preventing content overlap.

These features enhance accessibility, ensuring usability for all users.

Challenges and Lessons Learned

The Challenges I faced include

  1. Challenge: Designing a Consistent Layout Across Pages
    • Resolution: Ensured consistent layout by using a global CSS stylesheet and shared templates for all pages. Regularly tested across browsers to ensure uniformity.
    • Lesson Learned: Creating reusable components saves time and ensures consistency across pages.
  2. Challenge: Integrating Complex Navigation
    • Resolution: Initially faced issues with dropdown menus. Resolved by refining the navigation bar structure and using JavaScript for smoother interactions.
    • Lesson Learned: Testing early and frequently helps identify potential issues before they become bigger problems.
  3. Challenge: Mobile Responsiveness
    • Resolution: Struggled with elements not scaling well on mobile. Solved by using media queries and flexible grid layouts.
    • Lesson Learned: Always prioritize mobile-first design to ensure accessibility across all devices.
  4. Challenge: Content Formatting and Validation
    • Resolution: Encountered issues with input validation and content formatting. Resolved by implementing JavaScript validation and organizing content into manageable blocks.
    • Lesson Learned: Validation is crucial to avoid data entry issues and ensure smooth user interaction.
  5. Challenge: Time Management
    • Resolution: Balancing coursework and group work was challenging. Improved time management by setting clear milestones and communicating effectively with team members.
    • Lesson Learned: Breaking down tasks and setting deadlines within the group increases productivity and reduces stress.

Compliance

The pages comply with Janet regulations by adhering to accessibility standards, ensuring all users can access content regardless of ability. I implemented semantic HTML for better structure, used proper alt text for images, and ensured a logical heading hierarchy for screen readers. Navigation is keyboard-friendly with anchor links, supporting easy access for users with disabilities. Additionally, the design follows responsive practices, ensuring compatibility across devices. These steps align with Janet's emphasis on inclusive, accessible, and mobile-friendly web content for all users.
